Overview

In this project, we aim to create a realistic soap bubble simulation, capturing their dynamic motion under surface tension and rendering their iconic iridescent, rainbow-like appearance with ray tracing based techniques.

Bubble BSDF

In rendering the bubbles for this project, we extended the HW3 code to incorporate physically accurate thin-film transmission and reflection. Our first step was to implement a helper function thin_film_RT to compute the intensities of light that are reflected and refracted by the bubble’s surface. We specified the film’s refractive index 𝑛𝑓, its thickness 𝑑, and the refractive index of air 𝑛0. Using these parameters, along with the light’s wavelength λ and incident angle 𝜃0, we first applied Snell’s law to calculate the film angle 𝜃𝑓.

n0 · sin θ0 = nf · sin θf
cos θf = √ [ 1 − (n0/nf)2 · (1 − cos2θ0) ].

Phase thickness inside the film:

δ = (2π · nf · d · cos θf) / λ,

We then applied the following formulas to compute the Fresnel reflection (r) and transmission (t) amplitude coefficients for both s polarization (vertical) and p-polarization (parallel).

r01s = ( n0cos θ0 − nfcos θf ) / ( n0cos θ0 + nfcos θf )
r01p = ( nfcos θ0 − n0cos θf ) / ( nfcos θ0 + n0cos θf )
t01s = ( 2 n0cos θ0 ) / ( n0cos θ0 + nfcos θf )
t01p = ( 2 n0cos θ0 ) / ( nfcos θ0 + n0cos θf )

The f|3 case is analogous for r12⋅, t12⋅.

Finally, we applied the Airy closed-form solution to compute the final reflected and transmitted light intensities.

The total amplitude reflection and transmission for the thin film (phase factor eiδ) are:

rtot⋅ = [ r01⋅ + r12⋅ · ei·2δ ] / [ 1 + r01⋅ r12⋅ · ei·2δ ]
ttot⋅ = [ t01⋅ t12⋅ · ei·δ ] / [ 1 + r01⋅ r12⋅ · ei·2δ ].

Convert amplitude to power (intensity) and apply radiometric correction for transmission:

R⋅ = | rtot⋅ |2
T⋅ = ( n3cos θ3 / ( n0cos θ0 ) ) · | ttot⋅ |2.

For unpolarized light, take the s/p average:

R = ½ ( Rs + Rp ),    T = ½ ( Ts + Tp ).

After this the thin_film_RT can return the intensity ratio of the reflected light and the transmiited light compared to the incident light.

Delta BSDF Sampling

To ensure our bubble bsdf rendering returns the correct color, we construct an RGB wavelength channel and use the helper function we just implemented to obtain the reflectance and transmittance for each RGB wavelength. We then use sRGB weights to calculate the reflected intensity (wR) and transmitted intensity (wT) perceived by the human eye. We then calculate the light reflection probability pR = wR / (wT + wR) and use it to determine whether the sample returns reflected or transmitted light.

  • Reflection geometry: perfect mirror (local reflection).  frefl = ( Rrgb · pR ) / |cos θi|.
  • Transmission geometry: for a single-sided thin film, approximate as straight-through (local ωi = −ωo).  ftrans = ( Trgb · pT ) / |cos θi|.

Skybox

To enhance visual richness and avoid a monotonous background, we implemented a skybox so that bubbles could reflect and refract more varied and realistic colors instead of appearing in a blank scene. We loaded six images, each mapped to one face of the cube, forming a complete surrounding environment. In the view matrix, we set v(0,3) = v(1,3) = v(2,3) = 0to remove translation while preserving rotation, ensuring that the cube remains stationary relative to the camera and appears infinitely distant.

For path tracing, we created a PathtracingSkybox::sample method that takes a normalized direction vector as input, determines which cube face this direction intersects, converts it to 2D texture coordinates, and returns the corresponding pixel color from that face. This method enabling the path tracer to return accurate background colors when rays miss all geometry.
